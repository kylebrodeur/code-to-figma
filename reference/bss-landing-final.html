Zero Data Residency

We process your data.
We never store it.

Stateless transit architecture. Your spreadsheet data never touches a database, never gets logged, never creates GDPR liability.

RAM-Only Processing
Your JSON payload is received, transformed, and forwarded entirely in memory. The moment the response is sent, the data is gone. No write path to any database for payload data.
We Store Mapping Rules, Nothing Else
The only thing persisted is your column mapping schema — the structural instructions for how to rearrange data. A recipe, not the ingredients.
Zero Data Residency Risk
No GDPR Article 28 processor liability. No SOC 2 scope creep. If your legal team asks "where does the data live?" — the honest answer is: it doesn't.
OAuth2 Credential Isolation
Google Sheets tokens scoped per-user, never logged. Zapier webhook URLs stored encrypted at rest.

// What our inbound handler does
async function handleBulkInbound(req, res) {
  // 1. Load mapping rules only
  const mapping = await db.getMapping(req.mappingId);
  // 2. Payload lives ONLY in RAM
  const rows = req.body.data; // ← never written to DB
  // 3. Transform in memory
  const mapped = applyMapping(rows, mapping);
  // 4. Write to Sheets via OAuth
  await sheets.batchUpdate(mapped);
  // GC clears payload on function return
  return res.json({ tasks: 1, rows: mapped.length });
} // Zero persistence. Zero liability.
