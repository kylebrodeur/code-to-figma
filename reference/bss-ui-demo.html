08 Do / Don't
Do
One gold word per heading — exactly one
Martian Mono for all buttons, labels, nav, data
Rust for problems/errors, teal for success/after
Weight + colour for emphasis — no italic
Max 4px border-radius (2px for most components)
Gold for focus rings across all interactive elements
Min 44px touch targets on all interactive elements
CSS variables for every colour — no hardcoded hex
Don't
Multiple coloured words in one heading
Inter, DM Sans, Space Grotesk — wrong aesthetic
Italic — ever, in any context
Border-radius above 4px (no rounded cards)
Gold fills on non-primary actions
Colour as the only signal — pair with icon or shape
Animations without prefers-reduced-motion guard
Hardcoded colours — defeats theming
